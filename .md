## 参考 Superpowers 模式的优化

### Description 格式

遵循 Superpowers 的 `Use when [condition] - [what it does]` 格式：

```yaml
description: Use when the user mentions [触发条件] - [具体做什么]
```

例如：
- `Use when the user mentions connecting to or logging into the HNAICC cluster - configures SSH credentials, verifies key permissions, and tests connectivity.`
- `Use when the user mentions checking job status, queue utilization, or cluster info on the HNAICC cluster - covers cjobs, queue management, and job control commands.`

### Skill 内容结构

1. **Checklist** — 每个 skill 以任务清单开头，明确列出必须完成的步骤
2. **Prerequisites** — 标注依赖（主要是 SSH）
3. **Procedure steps** — 按顺序编号的步骤
4. **Reference tables** — 队列、参数、状态码等参考信息
5. **Common Pitfalls** — 常见问题和解决方案
6. **Next Steps** — 引导到下一个相关 skill

### 命名规范

- `-A` (Application name): 项目级标识，同项目共享
- `-J` (Job name): 作业级标识，每个作业唯一

### CPU 选择指导

aip_submit 中添加了决策流程：
1. 估算内存需求
2. 计算最小核心数: `ceil(memory_needed / 11GB)`
3. 向上取整到允许的值: 1, 2, 4, 8, 16, 32, 44, 64, 88
4. 考虑并行性
